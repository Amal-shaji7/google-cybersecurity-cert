# USB Drive Attack Vector Analysis

## Overview

A USB drive bearing the hospital's logo was found
in a parking lot. This activity involved safely
investigating the device contents using a virtualised
environment, analysing what an attacker could do
with the information found, and recommending
technical, operational, and managerial controls
to mitigate USB baiting attacks.

The scenario demonstrates how physical attack
vectors, often overlooked in favour of digital
threats and this can be just as damaging to an
organisation's security posture.

## Attack Type Covered

| Attack | Description |
|--------|-------------|
| USB Baiting | Planting a USB device in a location where a target is likely to find and plug it in — delivering malware or harvesting sensitive information |

---

## Control Categories Applied

| Category | Example Control |
|----------|----------------|
| Technical | Disable AutoPlay, endpoint media scanning |
| Operational | Removable media policy, antivirus scanning |
| Managerial | Security awareness training, incident reporting procedures |
