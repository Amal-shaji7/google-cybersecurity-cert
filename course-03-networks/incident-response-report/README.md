# Incident Response Report — ICMP Flood DoS Attack
## NIST Cybersecurity Framework (CSF) Analysis

## Overview

This folder documents an incident response report developed 
following a Denial of Service attack that disrupted the 
company's internal network for approximately two hours.

A multimedia company offering web design, graphic design, and 
social media marketing services experienced a Denial of Service 
attack that disrupted its entire internal network for 
approximately two hours. A malicious actor exploited an 
unconfigured firewall by sending a continuous flood of ICMP 
ping packets, overwhelming network capacity and preventing all 
internal traffic from accessing network resources. The incident 
management team responded by blocking ICMP traffic, taking 
non-critical services offline, and restoring critical services. 
This report applies the NIST Cybersecurity Framework v2.0 
across all six core functions to analyse the incident, document 
the response, and develop a structured security improvement 
plan to prevent recurrence.

> **Note:** The course material references five CSF functions. 
> This report applies all six functions including **Govern**, 
> added in NIST CSF v2.0 (2024), which provides the 
> organisational governance context underpinning the other five.
